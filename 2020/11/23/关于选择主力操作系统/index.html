关于选择主力操作系统思考

Windows、MacOS、Linux 每一个操作系统都有自己的很明显的优点，这点是无容置疑的。在竞争激烈的21世纪，存在即合理。没有必要说哪个操作系统好与坏，每个系统针对的人群都是有些区别的。特别是 windows ，有贬低的也有吹捧的。

贬低的认为他们已经厌倦了各种流氓软件之间的相互竞争，电脑越用越慢，缓存越用越多。容易中毒，碰到某些软件的弹窗甚至关不掉，浏览器主页莫名被修改，默认开启应用卸载后再开启文件会有残余。装360杀毒，殊不知360就是最大的病毒。

但 windows 也没有多么的不堪，也有很多优点。发展比较早，用户数量多，在适配、驱动和稳定性方面都是没得说（相对于 linux）。或者从另外一个角度说，有些人无法离开 windows。学化工的同学时常要用到的有限元分析软件Ansys，STAR-CCM+，COMSOL 还有流程模拟软件 ASPEN Plus，离开 windows 想都不要想。学机械的同学工程制图要用 CATIA，Creo CAD，也是不可能离开 windows 的。

linux 以其开源和可定制化出名。程序员能够完全免费的使用 linux，看到其内核代码，并且能够进行修改然后重新编译发布甚至发售。正常使用中你能够随心所欲的定制系统，上至桌面软件，下至常用指令。几乎所有的工作都可以用高效的指令操控，即使不在物理机旁，也只需一根网线 + SSH 就能完成大部分工作。日常使用来说 linux 无语伦比的安全性（相对于 windows）更是让人眼前一亮，linux 使用 root 权限控制将普通操作和管理操作区分开，linux 用户大部分时间都知道自己要做什么。但是 linux 有一个致命缺点，很多硬件兼容性不好，这主要由于 linux 用户不够多，没有那么多张着嘴要吃饭的孩子。

再说 macos，它和 mac 基本上是离不开的，如果要说稳定性，macos 首当其冲，因为大部分使用 macos 的电脑都是由 Apple 设计并生产的 mac 系列电脑，Apple 对于硬件和软件的控制能力是相当优秀的，特别是 2020 年秋季正式发布了 arm 架构自研处理器 M1 后其软硬件整合能力又达到了一个高度。macos 在程序员中的地位是神一般的，我个人认为是，它既有 windows 的稳定性还有 Linux 的可定制性（指和 Linux 相类似的终端），当然它也有一个特性：价格相对较高。有些同等配置的电脑 mac 的价格可能是它的3-4倍。

Windows日用，Linux部署项目，MacOS编程开发

折腾各种Linux衍生版本不叫学习，只能算浪费时间。《刻意练习》的思想是：有效而重复的努力，才能有好的结果。
