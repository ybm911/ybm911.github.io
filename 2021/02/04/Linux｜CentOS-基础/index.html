基础

目录结构

/bin
存放经常使用的命令
/sbin
super user，存放管理员使用的系统管理程序
/lib
系统动态链接库
/lost+found
系统非法关机后会将一些文件存放进来
/media
系统可自动识别的设备挂载目录
/opt
optional 额外安装的软件的目录
/proc（不能动）
process 系统内存的映射
/selinux
只有 Redhat/CentOS 有，存放 selinux （Linux 安全机制）的相关文件
/sys（不能动）
sysfs 文件系统总是被挂载在 /sys 挂载点上，sysfs 是 Linux 内核设计中较新的一种虚拟的基于内存的文件系统 sys 是设计出来用来改进 proc 的这种访问方式的
/usr
unix shared resources 共享资源，很多应用程序和文件的目录，类似于 program files
/var
variable 变量，存放经常被修改的文件，包括日志
/run
系统启动以来的信息
/srv（不能动）
系统启动之后需要提取的数据

RPM 与 YUM

rpm

rpm(RedHat Package Manager) RedHat 软件包管理工具，公认的行业标准。

rpm 包名基本格式：firefox-60.2.2-1.el7.centos.x86_64

名称+版本号+适用操作系统

i686、i386 表示32位系统，noarch 表示通用

yum

yum 基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包并且安装，可以自动解决依赖性关系，一次安装所有依赖的软件包。
